Visual Components

Visual components in Slice.js are the building blocks of your user interface. These components use the Web Components API, extending HTMLElement to create custom elements that can be used directly in your HTML markup.

What are Visual Components?

Visual components in Slice.js:

  • Render visible UI elements that users can interact with
  • Encapsulate HTML structure, CSS styling, and JavaScript behavior
  • Follow a consistent lifecycle pattern and property management
  • Can be composed together to build complex interfaces

Visual Component Structure

Each visual component consists of three key files:

  • ComponentName.js - The JavaScript class that defines behavior and properties
  • ComponentName.html - The HTML template that defines structure
  • ComponentName.css - The CSS styles specific to the component

Component Lifecycle

Visual components follow a standard lifecycle with two key async methods:

  1. constructor(props) - Initialize the component and set up event listeners
  2. async init() - Called once after construction to set default values and perform additional initialization
  3. async update() - Called every time the component is rendered, allowing you to refresh data and update the UI
  4. Property getters/setters - Manage the component's state and update the DOM accordingly

Key concepts for nesting components:

  • Async initialization: Use async/await in the init() method to ensure child components are fully built
  • Component communication: Pass data down to child components via props or use custom events for child-to-parent communication
  • DOM management: Store references to child components for later access and manipulation
  • Lifecycle coordination: Ensure parent updates trigger child updates when needed
