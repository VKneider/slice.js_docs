Configuration File: sliceConfig.json

Core Configuration

The sliceConfig.json file is the heart of your Slice.js application configuration. It defines how the framework's core components function, file paths, and enabled features. This file is located at the project root and is loaded automatically when initializing the application.

Configuration File Structure

Main Configurations

Each section of the configuration file controls a different aspect of the framework. Below, you'll find a detailed explanation of each configuration and its options.

Component Categories

The categories defined in paths.components determine the types of components you can create with the Slice.js CLI. Each category has a specific purpose and can contain different types of components.

AppComponents

AppComponents are application-specific components, generally composed of multiple visual components. These components form the main pages and sections of your interface.

  • High-level components that represent complete functionalities
  • Can contain business logic and visual components
  • Typically correspond to specific routes or sections of the application

Visual

Visual components are the fundamental building blocks of the user interface. They are reusable components that represent visual elements such as buttons, cards, inputs, etc.

  • Reusable UI components across the entire application
  • Contain HTML and CSS in addition to JavaScript code
  • Focused on presentation and basic interaction

Service

Service components encapsulate business logic and services without visual interface. They provide functionalities such as data management, API connections, storage, etc.

  • Components without user interface (no HTML or CSS)
  • Provide services and functionalities to other components
  • Manage communication with APIs, storage, and complex logic

Customizing Core Components

Customizing the Loading Component

The Loading component is displayed during navigation between routes when loading.enabled is activated. You can completely customize this component by modifying the files in src/Components/Visual/Loading/.

Creating Custom Themes

You can create custom themes by adding new CSS files in src/Themes/ and then configuring themeManager.defaultTheme to use your custom theme.

Updating the Configuration

Modifying the sliceConfig.json file allows you to completely customize Slice.js behavior to adapt it to your application's needs. Changes to this file will take effect the next time you start the application.

Best Practices

✓

Maintain Category Structure

Respecting the category structure (AppComponents, Visual, Service) improves code organization and maintainability.

✓

Enable the Debugger During Development

Activating debugger.enabled during development facilitates component inspection and testing.

✓

Customize Loading to Improve UX

Adapt the Loading component to match your application's aesthetics and improve user experience.

✓

Create Consistent Themes

When creating custom themes, make sure to define all necessary CSS variables to maintain visual consistency.

✓

Disable Logger in Production

Consider disabling logger.enabled in production to improve performance, or at least, deactivate info type logs.
