sliceConfig.json - Complete Configuration Guide

Core Configuration

The sliceConfig.json file is the central configuration hub of your Slice.js application. Located at the project root, this file controls framework behavior, component paths, development tools, theming, routing, and production optimizations. It's automatically loaded during application initialization and affects how all structural components operate.

Component Categories

The paths.components section defines different types of components you can create. Each category serves a specific purpose in your application architecture.

AppComponents

AppComponents are high-level, application-specific components that typically represent complete pages, sections, or major features. They orchestrate multiple Visual components and contain business logic.

  • Complete page or section components
  • Business logic and data management
  • Composition of multiple Visual components
  • Route-level components

Examples: HomePage, UserDashboard, ProductCatalog, CheckoutPage

Visual Components

Visual components are reusable UI building blocks that represent specific interface elements. They focus on presentation and user interaction without business logic.

  • Reusable UI elements
  • Consistent styling and behavior
  • Static Props support for configuration
  • Event handling and user interaction

Examples: Button, Input, Card, Modal, Dropdown, Grid

Service Components

Service components handle business logic, data operations, and API communication without any visual representation. They provide functionality that can be shared across Visual and AppComponents.

  • Data fetching and processing
  • Business logic implementation
  • API communication
  • Shared functionality across components

Examples: AuthService, ApiClient, DataValidator, CacheManager

Structural Components

Structural components are framework-level components that provide core functionality. These are not created by users but are part of the Slice.js framework itself.

  • Framework core functionality
  • Router, Controller, Logger, Debugger
  • Pre-initialized by the framework
  • Not user-creatable

Examples: Router, Controller, StylesManager, Logger, Debugger

Troubleshooting Configuration Issues

‚ùå Components not loading

Check that component paths in paths.components are correct and the directories exist. Ensure component names match exactly between configuration and file system.

‚ùå Themes not applying

Verify that themeManager.enabled is true and the theme name matches a CSS file in your themes directory. Check browser console for loading errors.

‚ùå Debugger not working

Ensure debugger.enabled is true and you're using the correct click type ("left" or "right"). The debugger only works in development mode.

‚ùå Routing issues

Check that your routes file exists at the specified path and exports a valid routes array. Verify the router.defaultRoute corresponds to an existing route.

Configuration Migration

When updating your Slice.js application, you may need to update your sliceConfig.json file to take advantage of enhanced features:

  • Static Props support: No configuration changes needed - works automatically with existing setup
  • Enhanced Router: Works with existing router configuration - no changes needed
  • Improved Debugger: No changes needed - enhanced debugging works with existing debugger configuration
  • Loading Component: Simple enable/disable configuration available
