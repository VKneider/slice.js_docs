Slice.js CLI Commands

Slice.js comes with a powerful command-line interface (CLI) that helps you create, manage, and run your Slice.js applications. These commands are defined in your package.json file and can be run using npm.

Initialize Project

The slice:init command sets up a new Slice.js project with the proper folder structure and configuration files. It's the first command you should run when starting a new Slice.js project.

Create Component

The slice:create command helps you create new components with the proper structure. It generates the JavaScript, HTML, and CSS files for your component.

List Components

The slice:list command shows you all available components in your project. This is useful for seeing what components you have and where they're located.

Delete Component

The slice:delete command allows you to safely remove components from your project. It ensures all files related to the component are properly deleted.

Start Server

The slice:start command starts the development server for your Slice.js application. It serves your application and provides an interactive server menu.

CLI Best Practices

  • Always run slice:init first when creating a new project
  • Use slice:create to ensure components follow the proper structure
  • Run slice:list when you need to reference existing components
  • Use slice:delete instead of manually deleting component files
  • Keep the server running with slice:start during development

Pro Tips

Multiple Terminals: For the best development experience, keep one terminal running slice:start and use another terminal for other commands.

Component Categories: When creating components, choose the appropriate category:

  • Visual: For UI components that users interact with
  • Service: For components that provide functionality but no UI
  • AppComponents: For application-specific components
