Slice.js CLI Commands

Slice.js comes with a powerful command-line interface (CLI) that helps you create, manage, and run your Slice.js applications. The CLI is organized into logical command groups for better workflow management.

Project Lifecycle Commands

Initialize Project

The slice:init command sets up a new Slice.js project with the proper folder structure and configuration files. It's the first command you should run when starting a new Slice.js project.

Development Server

The slice:dev or slice:start command starts the development server for your Slice.js application. It serves your application from the /src directory and provides live reloading.

Local Component Management

Create Component

The slice:component create command helps you create new local components with the proper structure. It generates the JavaScript, HTML, and CSS files for your component.

List Components

The slice:component list command shows you all available components in your local project. This is useful for seeing what components you have and where they're located.

Delete Component

The slice:component delete command allows you to safely remove components from your local project. It ensures all files related to the component are properly deleted.

Registry Component Management

Get Components

The slice:get command downloads and installs components from the official Slice.js repository. You can install multiple components at once and specify options like force overwrite or service components.

Browse Components

The slice:browse command shows you all available components in the official repository. This helps you discover new components and understand what's available for installation.

Sync Components

The slice:sync command updates all your local components to the latest versions from the repository. This ensures you have the most recent bug fixes and improvements.

Maintenance Commands

Version & Updates

The slice:version and slice:update commands help you check your current version and look for available updates for both the CLI and the Slice.js framework.

Complete CLI Workflow

When developing with Slice.js, you typically follow a comprehensive workflow that involves multiple command categories working together. Here's how all the commands work together:

CLI Best Practices

  • Always run slice:init first when creating a new project
  • Use slice:dev for development and keep it running during development
  • Use slice:component create to ensure components follow the proper structure
  • Use slice:get to install official components from the repository
  • Run slice:sync regularly to keep components updated
  • Use slice:build before deploying to production
  • Check for updates with slice:update periodically

Pro Tips

Command Groups: The CLI is organized into logical groups for better workflow management:

  • Project Lifecycle: init, dev, start
  • Local Components: component create, component list, component delete
  • Registry Operations: get, browse, sync
  • Production: build
  • Maintenance: version, update

Multiple Terminals: For the best development experience, keep one terminal running slice:dev and use another terminal for other commands.

Component Categories: When creating components, choose the appropriate category:

  • Visual: For UI components that users interact with (requires HTML, CSS, JS)
  • Service: For components that provide functionality but no UI (JS only)
  • AppComponents: For application-specific components (JS only)

Registry vs Local: Use slice:get for official components and slice:component create for custom components specific to your project.
