Structural Components

Structural components form the backbone of the Slice.js framework. They provide core functionality like component management, routing, styling, logging, and debugging. Unlike Visual or Service components, Structural components are typically pre-initialized by the framework and accessed through the global slice object.

Controller

The Controller manages all component instances, handles component registration, and provides the build() method for creating new components.

Key responsibilities:

  • Component registration and lifecycle management
  • Template and class loading
  • Component construction and property management
  • Component caching and referencing

Router

The Router handles navigation between different views in your application, supporting both history-based navigation and route parameters.

Key responsibilities:

  • Route registration and matching
  • History management and navigation
  • Dynamic route parameters
  • Integration with Route and MultiRoute components

Developer usage:

StylesManager

The StylesManager handles global styles and themes, allowing for dynamic theme switching and consistent styling across the application.

Key responsibilities:

  • Theme loading and application
  • Component style registration
  • Global style management
  • Theme persistence

Developer usage:

Logger

The Logger provides a consistent way to log information, warnings, and errors throughout your application.

Key responsibilities:

  • Error, warning, and info logging
  • Log filtering and retrieval
  • Structured logging with component context
  • Configurable output channels

Developer usage:

Debugger

The Debugger provides development tools for inspecting and manipulating components at runtime.

Key responsibilities:

  • Component property inspection
  • Live property editing
  • Debugging interface

Developer usage:

Configuration

Structural components are primarily configured through the sliceConfig.json file at the root of your project. This allows you to:

  • Enable or disable specific features
  • Set default options and behaviors
  • Configure paths and resource locations
  • Control logging, debugging, and theming behavior
