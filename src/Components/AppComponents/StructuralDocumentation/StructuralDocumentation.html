Structural Components

Structural components form the backbone of the Slice.js framework. They provide core functionality like component management, routing, styling, logging, and debugging. Unlike Visual or Service components, Structural components are typically pre-initialized by the framework and accessed through the global slice object.

Controller

The Controller manages all component instances, handles component registration, and provides the build() method for creating new components. Features the Static Props system for automatic default values and development-time validation.

Key responsibilities:

  • Component registration and lifecycle management
  • Template and class loading
  • Component construction and property management
  • Static Props system with automatic defaults
  • Automatic prop validation in development
  • Enhanced debugger integration
  • Component caching and referencing

Router

The Router handles navigation between different views in your application, supporting both history-based navigation and route parameters. Includes dynamic route management, performance improvements, and comprehensive lifecycle controls.

Key responsibilities:

  • Route registration and matching
  • History management and navigation
  • Dynamic route parameters
  • Navigation with anchor elements
  • Dynamic route management (add, remove, update routes)
  • Component pooling for better performance
  • Router statistics and monitoring
  • Lifecycle management (destroy, reinitialize)
  • Integration with Route and MultiRoute components

StylesManager

The StylesManager handles global styles and themes, allowing for dynamic theme switching and consistent styling across the application.

Key responsibilities:

  • Theme loading and application
  • Component style registration
  • Global style management
  • Theme persistence

Developer usage:

Logger

The Logger provides a consistent way to log information, warnings, and errors throughout your application.

Key responsibilities:

  • Error, warning, and info logging
  • Log filtering and retrieval
  • Structured logging with component context
  • Configurable output channels

Developer usage:

Debugger

The Debugger provides development tools for inspecting and manipulating components at runtime. Features Static Props integration, smart prop state detection, and anti-interference protection.

Key responsibilities:

  • Component property inspection
  • Static Props configuration display
  • Smart prop states (Used/Missing/Optional)
  • Default value visualization
  • Anti-interference protection
  • Live property editing with enhanced validation
  • Debugging interface with improved UX

Developer usage:

Configuration

Structural components are primarily configured through the sliceConfig.json file at the root of your project. This allows you to:

  • Enable or disable specific features
  • Set default options and behaviors
  • Configure paths and resource locations
  • Control logging, debugging, and theming behavior

Note: All features maintain 100% backward compatibility. Existing configurations continue to work while enhanced features are available as opt-in improvements.
