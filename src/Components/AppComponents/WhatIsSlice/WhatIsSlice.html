What is Slice.js?

Slice.js is a lightweight, component-based web framework built on vanilla JavaScript that embraces modern web standards. It provides a modular approach to building web applications with reusable components while maintaining the simplicity and performance of native browser technologies.

Designed for developers who appreciate clean architecture and minimal dependencies, Slice.js offers powerful features without introducing unnecessary complexity or bloat.

Slice.js Architecture

Slice.js is built on a three-layered architecture that separates concerns and promotes maintainable code:

Visual Components

UI Elements

Structural Components

Core Framework

Service Components

Business Logic

Visual Components

Build the user interface of your application. These components render HTML, handle user interactions, and manage UI state. Examples include buttons, inputs, cards, and navigation elements.

Button.js, Card.js, Input.js

Structural Components

Form the backbone of the framework. These components manage the application lifecycle, routing, component registration, and other core functionality. They're mostly internal but can be customized if needed.

Router.js, Controller.js, StylesManager.js

Service Components

Handle business logic, data processing, and external integrations. These components provide services to visual components and often interact with APIs, databases, or browser APIs.

FetchManager.js, IndexedDbManager.js, LocalStorageManager.js

Data Flow

Data in Slice.js flows in a predictable manner:

  1. User interacts with Visual Components
  2. Visual Components call Service Components for data or business logic
  3. Service Components process data and return results
  4. Visual Components update the UI based on the results
  5. Structural Components manage the lifecycle and coordination

Component Types

Visual Components

These components build your user interface and extend the HTMLElement class, leveraging the Web Components standard.

export default class Button extends HTMLElement {
  constructor(props) {
    super();
    slice.attachTemplate(this);
    this.$value = this.querySelector('.slice_button_value');
    
    slice.controller.setComponentProps(this, props);
  }

  get value() {
    return this._value;
  }

  set value(value) {
    this._value = value;
    this.$value.textContent = value;
  }
}

Service Components

These components handle business logic, data processing, and external integrations.

export default class FetchManager {
  constructor(props) {
    const { baseUrl, timeout } = props;
    this.baseUrl = baseUrl;
    this.timeout = timeout || 10000;

  }

Component Lifecycle

Slice.js components have a simple lifecycle that gives you control over their behavior:

  • constructor: Initialize properties and set up event listeners
  • init: Called after the component is created, good for async operations
  • render (optional): Update the component's DOM based on props
  • update (optional): Refresh the component when props change

Getting Started with Slice.js

Creating a Component

Here's how to create a simple counter component in Slice.js:

Counter.js

export default class Counter extends HTMLElement {
  constructor(props) {
    super();
    slice.attachTemplate(this);
    
    this.$count = this.querySelector('.count');
    this.$increment = this.querySelector('.increment');
    this.$decrement = this.querySelector('.decrement');
    
    this.$increment.addEventListener('click', () => this.increment());
    this.$decrement.addEventListener('click', () => this.decrement());
    
    slice.controller.setComponentProps(this, props);
    this.debuggerProps = ['count'];
  }
  
  init() {
    if (this._count === undefined) {
      this.count = 0;
    }
  }
  
  get count() {
    return this._count;
  }
  
  set count(value) {
    this._count = value;
    this.$count.textContent = value;
  }
  
  increment() {
    this.count++;
  }
  
  decrement() {
    this.count--;
  }
}

Using the Component

Once defined, you can use your component like this:

// Create an instance of the counter
const counter = await slice.build('Counter', {
  count: 5 // Initial value
});

// Add it to a parent component or container
const container = slice.getComponent('MainContainer');
container.appendChild(counter);

// Or use it in another component's template
const mySection = this.querySelector('.my-section');
mySection.appendChild(counter);

// Later, you can access or modify properties
console.log(counter.count); // 5
counter.count = 10;

Why Choose Slice.js?

Lightweight

Minimal footprint without the bloat of larger frameworks, leading to faster load times and better performance.

No Build Step

Develop directly with vanilla JavaScript, HTML, and CSS without requiring transpilers, bundlers, or complex build systems.

Easy Learning Curve

If you know basic HTML, CSS, and JavaScript, you can start building with Slice.js right away. No new template syntax to learn.

Standards-Based

Built on Web Components and other modern browser standards, ensuring compatibility and future-proofing your applications.

Component Library

Comes with pre-built, customizable UI components to accelerate development and maintain consistency.

Debugging Tools

Built-in tools for inspecting and debugging components, making development easier and more efficient.
