The Build Method

Core API

The build method is the cornerstone of Slice.js component creation. It's an asynchronous method that instantiates and configures components, handling all of the necessary resource loading, template attachment, and initialization.

Syntax

await slice.build(componentName, props);

Parameters

  • componentName
    string
    The name of the component to build as registered in components.js
  • props
    object
    (Optional) An object containing properties to set on the component

Return Value

A Promise that resolves to the constructed component instance, or null if construction fails.

Build Lifecycle

When slice.build() is called, the following steps occur:

  1. 1
    Verify that the component exists in the component registry
  2. 2
    Load the component's template (HTML) if not already cached
  3. 3
    Load the component's class (JS) if not already cached
  4. 4
    Load the component's styles (CSS) if not already cached
  5. 5
    Create an instance of the component with the provided properties
  6. 6
    Call the component's init() method if it exists
  7. 7
    Attach debug mode if enabled in configuration
  8. 8
    Register the component in the controller's active components registry
  9. 9
    Return the fully initialized component instance

Best Practices

  • ✓ Always use await with slice.build() since it returns a Promise
  • ✓ Provide a unique sliceId for components that need to be referenced later
  • ✓ Implement proper error handling around component building
  • ✓ Use the component's init() method for additional setup rather than the constructor
  • ✓ Remember that component properties are set after construction but before initialization
