LifeCycle Methods in Slice.js

Slice.js components have a well-defined lifecycle that allows you to hook into different phases of a component's existence. Understanding these lifecycle methods is crucial for proper component management, especially when working with routing and dynamic content.

Overview

Every Slice.js component goes through several phases during its lifetime:

  • Creation - When the component instance is first created
  • Initialization - When the component is set up and ready to be used
  • Updates - When the component needs to refresh its state or DOM
  • Destruction - When the component is no longer needed (handled automatically)

LifeCycle Flow

The init() Method

The init() method is called once after the component is created and its template is attached. This is where you should perform initial setup, fetch data, create child components, and establish event listeners.

When to use init()

  • Initial data fetching
  • Setting up event listeners
  • Creating and configuring child components
  • One-time DOM manipulations
  • Establishing component state

The update() Method

The update() method is called every time the component needs to refresh itself. This happens when the router re-uses cached components during navigation, or when you manually trigger updates.

When to use update()

  • Refreshing data that might have changed
  • Re-applying styles or DOM modifications
  • Updating component state based on new props
  • Synchronizing with external state changes

Component Caching

When you navigate between routes, the MultiRoute component caches previously created components. This means:

  • First visit: constructor → init() → component ready
  • Return visits: update() → component refreshed
