Themes in Slice.js

Slice.js includes a powerful theming system that allows you to easily customize the appearance of your entire application. Slice.js themes are based on CSS variables that control colors, fonts, borders, and other visual styles.

How the Theme System Works

The Slice.js theme system uses the ThemeManager, a structural component that manages the application and runtime theme switching. Themes are defined as separate CSS files located in the /src/Themes/ folder.

Default Themes

Slice.js comes with three built-in themes:

  • Light: A light theme with blue as the primary color.
  • Dark: A dark theme that reduces eye strain.
  • Slice: The default Slice.js theme.

Theme Configuration

You can configure the theme behavior in the sliceConfig.json file:

The configuration options are:

  • enabled: Enables or disables the theme system.
  • defaultTheme: Sets the default theme to be applied when the application starts.
  • saveThemeLocally: If enabled, saves the user's theme preference in localStorage.
  • useBrowserTheme: If enabled, uses the user's system preference (light/dark).

Best Practices for Using Themes

When developing components for Slice.js, follow these practices to ensure they work correctly with the theme system:

  • Always use the CSS variables defined in the themes instead of hardcoded colors.
  • For elements with different states (hover, active, etc.), create CSS rules that respect the theme's color scheme.
  • Test your components with different themes to ensure they look good in all of them.
  • If your component needs specific styles based on the current theme, you can add classes based on the current theme.

Theme Transitions

When changing from one theme to another, styles are updated immediately. If you want to add smooth transitions when changing themes, you can add CSS transition rules to your elements: 

* {
  transition: background-color 0.3s ease, color 0.3s ease;
}
